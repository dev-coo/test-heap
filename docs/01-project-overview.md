# 프로젝트 개요서: AlgoViz (알고리즘 시각화 플랫폼)

## 1. 비전

**"코드를 읽기 전에 눈으로 이해한다"**

알고리즘을 텍스트와 코드로만 배우면 동작 원리를 직관적으로 파악하기 어렵다. AlgoViz는 모든 주요 알고리즘을 단계별 애니메이션으로 시각화하여, 학습자가 각 단계에서 "왜 이 동작이 일어나는지"를 직접 보고 이해할 수 있게 한다.

## 2. 대상 사용자

| 사용자 유형 | 니즈 | 우리의 해법 |
|---|---|---|
| CS 전공 학생 | 자료구조/알고리즘 수업 보충 학습 | 교과서 순서에 맞춘 알고리즘 목록 |
| 코딩 테스트 준비생 | 알고리즘 동작 빠른 복습 | 핵심만 보여주는 간결한 UI |
| 비전공 입문자 | 프로그래밍 기초 개념 이해 | 한국어 설명, 낮은 진입 장벽 |
| 강의자/튜터 | 수업 중 시각 자료 | 전체 화면 모드, 느린 속도 |

## 3. 핵심 가치

### 3.1 단계별 추적 가능성
모든 알고리즘은 "스텝" 단위로 분해된다. 사용자는 각 스텝에서:
- **무엇이 비교/이동/방문되는지** 시각적으로 확인
- **왜 그 동작이 필요한지** 한국어 설명으로 이해
- **의사코드의 어느 줄이 실행 중인지** 하이라이트로 추적

### 3.2 능동적 학습
- 사용자가 직접 데이터를 입력하고 결과를 관찰
- 랜덤 데이터, 최선/최악 케이스 프리셋 제공
- 속도 조절, 일시정지, 한 스텝씩 진행 지원

### 3.3 비교를 통한 인사이트
- 유사 알고리즘 간 성능 비교 (버블 vs 퀵, BFS vs DFS 등)
- 같은 입력에 대한 연산 횟수 차이를 실시간 시각화
- 복잡도 이론과 실제 동작의 연결

## 4. 현재 상태 (프로토타입)

MinHeap 시각화 페이지가 완성되어 있으며, 다음 패턴이 검증되었다:

| 검증된 패턴 | 구현 상태 |
|---|---|
| Step 기반 애니메이션 모델 | `HeapStep { type, indices, description, heap }` |
| 이중 뷰 (배열 + 트리) | ArrayView + TreeView |
| 단계별 설명 패널 | StepInfo 컴포넌트 |
| 알고리즘 비교 뷰 | ComparisonView (정렬 배열 vs 힙) |
| 함수별 설명 + 미니 데모 | FunctionExplainer + MiniVisualization |
| 애니메이션 속도 조절 | 슬라이더 200ms~1500ms |
| 다크 테마 UI | CSS 변수 기반 |

## 5. 프로젝트 범위

### In Scope (포함)
- 30+ 알고리즘/자료구조 시각화
- 4개 카테고리: 정렬, 자료구조, 그래프, 탐색/기타
- 알고리즘별 독립 페이지 + 비교 기능
- 반응형 웹 (데스크톱 우선, 태블릿 지원)
- 한국어 UI

### Out of Scope (제외)
- 코드 에디터/실행 환경
- 사용자 계정/진행 상황 저장
- 모바일 네이티브 앱
- 영어 다국어 지원 (초기)

## 6. 성공 지표

| 지표 | 목표 |
|---|---|
| 알고리즘 커버리지 | Phase 1~2 완료 시 18개 알고리즘 |
| 페이지 로드 | 3초 이내 (코드 스플리팅 적용) |
| 빌드 에러 | 0 (TypeScript strict + ESLint 통과) |
| 반응형 | 768px 이상 모든 뷰 정상 동작 |

## 7. 기술 스택

| 영역 | 기술 | 이유 |
|---|---|---|
| 프레임워크 | React 19 | 이미 검증됨, 컴포넌트 기반 |
| 언어 | TypeScript (strict) | 타입 안전성, Step 모델 강제 |
| 빌드 | Vite 7 | 빠른 HMR, 코드 스플리팅 |
| 애니메이션 | Framer Motion | 이미 검증됨, 선언적 API |
| 라우팅 | React Router v7 | 카테고리/알고리즘별 URL |
| 그래프 시각화 | SVG 직접 렌더링 | 힙 TreeView에서 검증됨 |
| 스타일 | CSS Modules 또는 CSS 변수 | 컴포넌트 격리, 테마 지원 |
| 상태 관리 | useState/useRef (로컬) | 외부 라이브러리 불필요 |

## 8. 명명 규칙

- **프로젝트 이름**: AlgoViz (또는 사용자 결정)
- **URL 패턴**: `/sorting/bubble-sort`, `/data-structures/heap`, `/graph/bfs`
- **컴포넌트 파일**: PascalCase (`BubbleSort.tsx`)
- **알고리즘 로직**: `src/algorithms/{category}/{name}.ts`
- **공유 컴포넌트**: `src/components/shared/`
- **페이지 컴포넌트**: `src/pages/{category}/{Name}Page.tsx`
