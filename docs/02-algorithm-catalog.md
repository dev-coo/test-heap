# 알고리즘 카탈로그

## 범례

- **난이도**: 구현 난이도 (시각화 로직 + 뷰 복잡도 기준)
- **뷰 타입**: 해당 알고리즘에 필요한 시각화 컴포넌트
- **비교 대상**: 같은 페이지에서 비교할 알고리즘
- **Phase**: 개발 우선순위 단계

## 1. 정렬 알고리즘 (Sorting)

정렬은 알고리즘 학습의 출발점이다. 배열 뷰 하나로 대부분 시각화 가능하며, 서로 비교하기 좋다.

| # | 알고리즘 | 난이도 | 시간 복잡도 | 뷰 타입 | 비교 대상 | Phase |
|---|---------|--------|-----------|---------|----------|-------|
| 1 | **Bubble Sort** | ★☆☆ | O(n²) | 배열 | Selection, Insertion | 1 |
| 2 | **Selection Sort** | ★☆☆ | O(n²) | 배열 | Bubble, Insertion | 1 |
| 3 | **Insertion Sort** | ★☆☆ | O(n²) | 배열 | Bubble, Selection | 1 |
| 4 | **Merge Sort** | ★★☆ | O(n log n) | 배열 + 분할 트리 | Quick Sort | 2 |
| 5 | **Quick Sort** | ★★☆ | O(n log n) avg | 배열 + 파티션 | Merge Sort | 2 |
| 6 | **Heap Sort** | ★★☆ | O(n log n) | 배열 + 힙 트리 | Merge, Quick | 3 |
| 7 | **Counting Sort** | ★★☆ | O(n + k) | 배열 + 카운트 배열 | Radix | 3 |
| 8 | **Radix Sort** | ★★☆ | O(d(n + k)) | 배열 + 버킷 | Counting | 4 |

### 정렬 공통 시각화 요소
- **바 차트 뷰**: 배열 값을 높이로 표현한 막대 그래프 (핵심)
- **배열 인덱스 뷰**: 기존 힙 ArrayView와 유사
- **비교 카운터**: 비교 횟수, 교환 횟수 실시간 표시
- **최선/최악/평균 프리셋**: 이미 정렬된 배열, 역순 배열, 랜덤 배열

### 각 알고리즘 핵심 시각화 포인트

**Bubble Sort**
- 인접 두 원소 비교 → 교환을 반복하며 큰 값이 오른쪽으로 "버블링"
- 시각화: 비교 중인 두 원소 하이라이트, 정렬 완료된 오른쪽 영역 구분
- 학습 포인트: 왜 O(n²)인지 — 패스마다 하나씩 확정, n-1번 패스

**Selection Sort**
- 매 패스에서 최소값을 찾아 앞으로 이동
- 시각화: 현재 최소값 후보 하이라이트, 탐색 범위 표시
- 학습 포인트: 비교 횟수는 항상 n(n-1)/2 — 최선/최악 동일

**Insertion Sort**
- 현재 원소를 정렬된 부분에서 올바른 위치에 삽입
- 시각화: 정렬된 영역 / 미정렬 영역 구분선, 삽입 위치 찾기
- 학습 포인트: 거의 정렬된 데이터에서 O(n)으로 빠름

**Merge Sort**
- 재귀적 분할 → 병합
- 시각화: 분할 트리 (어떻게 쪼개지는지), 병합 과정 (두 배열 → 하나)
- 학습 포인트: 분할 정복의 대표, 안정 정렬, 추가 공간 O(n)

**Quick Sort**
- 피벗 선택 → 파티션 → 재귀
- 시각화: 피벗 강조, 파티션 과정 (작은 값 왼쪽, 큰 값 오른쪽)
- 학습 포인트: 피벗 선택이 성능을 결정, 불안정 정렬

## 2. 자료구조 (Data Structures)

자료구조는 데이터를 담는 "그릇"의 동작을 이해하는 것이 핵심이다.

| # | 자료구조 | 난이도 | 주요 연산 | 뷰 타입 | Phase |
|---|---------|--------|----------|---------|-------|
| 9 | **Stack** | ★☆☆ | push, pop, peek | 스택 뷰 | 1 |
| 10 | **Queue** | ★☆☆ | enqueue, dequeue, peek | 큐 뷰 | 1 |
| 11 | **MinHeap** | ★★☆ | insert, extractMin, peek | 배열 + 트리 | ✅ 완료 |
| 12 | **MaxHeap** | ★☆☆ | insert, extractMax, peek | 배열 + 트리 | 2 |
| 13 | **BST** | ★★☆ | insert, delete, search | 트리 | 2 |
| 14 | **AVL Tree** | ★★★ | insert, delete, rotation | 트리 + 회전 | 4 |
| 15 | **Hash Table** | ★★☆ | put, get, delete | 테이블 + 체이닝/프로빙 | 3 |
| 16 | **Linked List** | ★★☆ | insert, delete, search | 노드 체인 | 2 |
| 17 | **Deque** | ★☆☆ | pushFront/Back, popFront/Back | 큐 뷰 변형 | 3 |

### 각 자료구조 핵심 시각화 포인트

**Stack**
- 세로 쌓기 뷰: 접시 쌓듯이 위에서 push/pop
- 시각화: 새 원소가 위에 올라가고, pop 시 위에서 제거
- 학습 포인트: LIFO, 재귀 호출 스택과의 연결
- 활용 예시: 괄호 매칭, 실행 취소(Undo)

**Queue**
- 가로 줄서기 뷰: 왼쪽에서 dequeue, 오른쪽에서 enqueue
- 시각화: 원소가 뒤에 추가되고, 앞에서 제거
- 학습 포인트: FIFO, BFS에서의 활용
- 활용 예시: BFS, 프린터 대기열

**BST (이진 탐색 트리)**
- 트리 뷰: 노드 삽입 시 탐색 경로 하이라이트
- 시각화: 비교하며 내려가는 경로, 삭제 시 후속자 찾기
- 학습 포인트: 탐색 O(log n) vs O(n) 퇴화, 균형의 중요성
- 연산별 시나리오: 삭제 3가지 경우 (리프, 자식 1개, 자식 2개)

**Hash Table**
- 버킷 배열 + 체이닝 리스트 또는 프로빙 경로
- 시각화: 해시 함수 계산 과정, 충돌 발생 시 처리
- 학습 포인트: 해시 함수, 충돌 해결 (체이닝 vs 개방 주소법)

**Linked List**
- 노드 + 화살표 체인 뷰
- 시각화: 포인터 변경 애니메이션 (삽입/삭제 시)
- 학습 포인트: 배열 vs 연결 리스트 트레이드오프

## 3. 탐색 알고리즘 (Search)

| # | 알고리즘 | 난이도 | 시간 복잡도 | 뷰 타입 | 비교 대상 | Phase |
|---|---------|--------|-----------|---------|----------|-------|
| 18 | **Linear Search** | ★☆☆ | O(n) | 배열 | Binary Search | 1 |
| 19 | **Binary Search** | ★☆☆ | O(log n) | 배열 + 범위 | Linear Search | 1 |

### 시각화 포인트

**Linear Search**
- 시각화: 왼쪽부터 하나씩 확인, 찾은 원소 강조
- 학습 포인트: 단순하지만 비효율적, 정렬 불필요

**Binary Search**
- 시각화: 탐색 범위(low~high) 축소 과정, mid 계산, 절반씩 제거
- 학습 포인트: 정렬된 배열 전제, log₂n번 만에 탐색
- 비교: 같은 데이터에서 Linear vs Binary 탐색 횟수 차이

## 4. 그래프 알고리즘 (Graph)

그래프는 시각화의 가치가 가장 큰 영역이다. 텍스트만으로는 이해가 매우 어렵기 때문이다.

| # | 알고리즘 | 난이도 | 시간 복잡도 | 뷰 타입 | 비교 대상 | Phase |
|---|---------|--------|-----------|---------|----------|-------|
| 20 | **BFS** | ★★☆ | O(V+E) | 그래프 + 큐 | DFS | 2 |
| 21 | **DFS** | ★★☆ | O(V+E) | 그래프 + 스택 | BFS | 2 |
| 22 | **Dijkstra** | ★★★ | O((V+E)log V) | 그래프 + 거리 테이블 | Bellman-Ford | 3 |
| 23 | **Bellman-Ford** | ★★☆ | O(VE) | 그래프 + 거리 테이블 | Dijkstra | 3 |
| 24 | **Kruskal (MST)** | ★★★ | O(E log E) | 그래프 + Union-Find | Prim | 4 |
| 25 | **Prim (MST)** | ★★★ | O(E log V) | 그래프 + 우선순위 큐 | Kruskal | 4 |
| 26 | **Topological Sort** | ★★☆ | O(V+E) | DAG | — | 4 |

### 그래프 공통 시각화 요소
- **그래프 뷰**: 노드(원) + 간선(선), 드래그로 노드 위치 조정 가능
- **인접 리스트/행렬 뷰**: 그래프의 데이터 표현
- **방문 상태 색상**: 미방문(회색), 탐색 중(주황), 방문 완료(초록)
- **보조 자료구조 뷰**: 큐(BFS), 스택(DFS), 거리 테이블(Dijkstra)

### 각 알고리즘 핵심 시각화 포인트

**BFS (너비 우선 탐색)**
- 시각화: 레벨별로 파도처럼 퍼져나가는 탐색, 큐의 변화
- 학습 포인트: 최단 경로 보장 (가중치 없는 그래프), 레벨 순서

**DFS (깊이 우선 탐색)**
- 시각화: 한 방향으로 깊이 내려갔다가 백트래킹, 스택의 변화
- 학습 포인트: 경로 탐색, 사이클 감지, 재귀와의 관계

**Dijkstra**
- 시각화: 시작 노드에서 거리 확산, 우선순위 큐에서 최소 거리 노드 선택
- 학습 포인트: 그리디 접근, 음수 가중치 불가 이유

## 6. 동적 프로그래밍 (Dynamic Programming)

| # | 알고리즘 | 난이도 | 뷰 타입 | Phase |
|---|---------|--------|---------|-------|
| 31 | **Fibonacci** | ★☆☆ | 재귀 트리 + 메모 테이블 | 3 |
| 32 | **Knapsack** | ★★★ | DP 테이블 | 4 |
| 33 | **LCS** | ★★★ | DP 테이블 + 역추적 | 4 |
| 34 | **Coin Change** | ★★☆ | DP 배열 | 4 |

### 시각화 포인트
- **재귀 트리 vs 메모이제이션**: 같은 문제의 재귀 호출 트리에서 중복 계산을 시각적으로 보여주고, 메모이제이션으로 어떻게 줄어드는지 비교
- **DP 테이블 채우기**: 셀이 하나씩 채워지는 애니메이션
- **역추적**: 최적 해를 역으로 따라가는 경로

## Phase별 알고리즘 요약

### Phase 1 (기초 — 7개)
> 배열 기반, 구현 단순, 시각화 패턴 확립

Bubble Sort, Selection Sort, Insertion Sort, Linear Search, Binary Search, Stack, Queue

### Phase 2 (중급 — 9개)
> 트리/재귀/그래프 도입, 공유 컴포넌트 확장

Merge Sort, Quick Sort, MaxHeap, BST, Linked List, BFS, DFS, Tree Traversals (In/Pre/Post/Level)

### Phase 3 (심화 — 7개)
> 고급 자료구조, 고급 그래프

Heap Sort, Counting Sort, Hash Table, Deque, Dijkstra, Bellman-Ford, Fibonacci DP

### Phase 4 (전문 — 8개)
> 복잡한 알고리즘, 전문 영역

Radix Sort, AVL Tree, Kruskal, Prim, Topological Sort, Knapsack DP, LCS DP, Coin Change DP
